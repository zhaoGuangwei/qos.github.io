  • config-root-ca

root CA用于校验QSC和QCP，不存在相关业务时可不配置。CA的获取和使用请查阅CA 文档

使用qosd config-root-ca初始化root CA公钥到配置文件。

$ qosd config-root-ca --qcp <qcp-root.pub> --qsc <qsc-root.pub>
1

更多操作说明查看设置CA

查看genesis.json内容，确认配置成功。

  • create-validator

使用qosd gentx和qosd collect-gentxs初始化validator到配置文件中，只有配置了validator才能正常运行QOS网络。

使用上面的初始化账户地址作为owner

$ qosd gentx --moniker validatorName --owner qosacc1hqcz9hhxa7qqxghc276vxxgcd3qkr279nz5gfq --tokens 10
1

主要参数说明:

  • --owner 操作者账户地址
  • --moniker 验证节点名字
  • --logo logo
  • --website 网址
  • --details 详细描述信息
  • --tokens 绑定tokens，不能大于操作者持有QOS数量
  • --compound 收益复投方式，默认false，即收益不复投

更多操作说明参照生成创世交易

运行：

$ qosd collect-gentxs
1

将创建验证节点交易写入genesis.json文件中。

  • start
$ qosd start --log_level debug
1

如果一切正常，会看到控制台输出打块信息

# Cluster

  • qosd testnet qosd-testnet命令可以批量生成一个测试网络多个验证节点配置信息

假设第一台机器IP: 192.168.1.100

$ qosd testnet --v 4 --name capricorn --starting-ip-address 192.168.1.100
Successfully initialized 4 node directories
1
2
3

会在当前目录下生成mytestnet文件夹，分别放置node0-3配置文件。 其中priv_validator_owner.json为对应validator owner私钥，可通过qoscli keys import导入。

  • start 启动前请确保按照安装说明在四台机器上正确安装QOS。 拷贝node0-3至不同机器，分别执行：
$ qosd start --home <directory_for_config_and_data>
